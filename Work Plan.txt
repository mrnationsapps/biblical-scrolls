 ① Build the cast (← we are here)
  Complete the canonical roster era-by-era (Genesis done through ch. 50), then sweep up the deferred genealogy-table
  blocks, then inject fictional NPCs per era to populate the world. Output: characters.csv.

  ② Build the world structure
  - The Beat map — divide the whole Bible into narrative units with density weights (the pacing engine from DESIGN
  §5.2). This is what turns "4,000 years" into "one calendar year."
  - The social graph — who's friends/family/enemies with whom (we're already seeding this via Family_Links + Faction).
  - Story threads — the drama arcs, captured in the new doc (see below).

  ③ Generate the content (the biggest stage)
  Per Beat: posts, trickle-in comment threads, news reports, ads, life events, deaths/memorials — voice-tuned per
  character. I generate → you approve → it's scheduled into the database with release windows. This is the engine that
  actually fills the feed.

  ④ Build the backend
  Supabase/Postgres: the entity tables (DESIGN §7.2), the global world-clock, the content-release scheduler, push
  notifications, the feed algorithm + per-user "seen" tracking.

  ⑤ Build the app
  Expo/React Native: feed, profiles, friends/PYMK, reactions, comment threads, the world-clock header, memorial pages,
  news pages, ads.

  ⑥ Launch
  Seed season-one content, beta test, ship to the App Store / Play Store.
